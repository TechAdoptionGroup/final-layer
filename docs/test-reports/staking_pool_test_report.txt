WHAT EACH TEST PROVES
======================================================

T0-Bootstrap [PASS]
  First depositor does NOT inherit validator pre-existing locked
  balance as instant rewards. The contract syncs last_locked_balance
  from env::account_locked_balance() when total_stake_shares == 0.
  Without fix: first user could appear to own validator's billions.
  Fix in: deposit_and_stake(), if self.total_stake_shares == 0 { ... }

T1-DepositFee [PASS]
  Deposit fee (10 bps = 0.1%) is deducted before staking.
  1000 FLC deposit => 999 FLC staked (1 FLC fee).
  Confirms deposit_fee_bps parameter is correctly applied.

T2-LockupReject [PASS]
  Unstake is rejected within LOCKUP_NS (20s) of last stake action.
  deposit_and_stake, compound, unstake, claim_rewards all reset timer.
  Until unlock_timestamp_ns passes, unstake/unstake_all must fail.

T3-RewardAccrual [PASS]
  Staked balance increases over 3 epochs as validator earns rewards.
  internal_ping() propagates locked-balance increases on each tx.
  Even small stake sees positive growth.

T4-Compound [PASS]
  compound() reinvests rewards into principal AND resets lockup.
  staked_balance >= pre-compound, unlock_timestamp_ns refreshed.
  Recommended auto-reinvest mechanism.

T5-AddStake [PASS]
  Second deposit increases staked_balance by (deposit - fee).
  500 FLC => ~499.5 FLC increase (0.1% fee). Timer resets.

T6-ClaimRewards [PASS]
  claim_rewards() transfers earned rewards minus claim fee (10 bps).
  Requires lockup to have expired first.

T7-PartialUnstake [PASS]
  unstake(amount) moves funds to unstaked_balance for unbonding.
  Staked shares burned correctly, unstaked_balance increases.

T8-PrematureWithdraw [PASS]
  withdraw_all() immediately after unstake is REJECTED.
  4-epoch (NUM_EPOCHS_TO_UNLOCK=4) unbonding must elapse.
  Prevents instant stake-earn-withdraw abuse.

T9-FullUnstake [PASS]

  A sub-FLC dust residue (<1 FLC) is normal from share-math double rounding:

  Unstaked balance correctly reflects the full withdrawal amount.

======================================================
ROUNDING DUST vs. BOOTSTRAP QUIRK
======================================================

1. ROUNDING DUST (harmless, expected)
   Share math uses integer floor division throughout:
     shares = amount * total_shares / total_balance  (floor)
     amount = shares * total_balance / total_shares  (floor)
   Tiny yocto remainders stay in the pool as epsilon share-price lift.
   Acceptable as long as no user gets MORE than their fair share.
   Dust < 1000 yocto is essentially 0 FLC.

2. BOOTSTRAP QUIRK (serious — FIXED in this contract)
   Without fix:
     - last_locked_balance initialised to 0
     - First depositor triggers internal_ping()
     - reward = current_locked (e.g. 300M FLC) - 0 = 300M FLC
     - First depositor credited with 300M FLC instant "reward"
   With fix (confirmed by T0):
     if self.total_stake_shares == 0 {
         self.last_locked_balance = env::account_locked_balance();
     }
     Pre-existing locked balance is the baseline. Only new
     post-initialization increases count as delegator rewards.

T0 PASSED: first depositor saw ~999 FLC, not validator's locked billions.
T11 PASSED: residue after full unstake+withdraw cycle was ~0.000295 FLC
  (~295 * 10^18 yocto). This is a share-math rounding artifact:
  unstake_all() does two floor divisions (amount_for_shares, then shares_for)
  which can leave up to 1 share = tiny sub-FLC residue in the pool.
  This is NOT the bootstrap quirk — it is normal integer-division dust.
